## Contents【相关介绍】


在写项目 或 阅读别人的代码（一些优秀的源码）中，总能发现一些让你喜欢解决方式的代码**「这里的喜欢不是说你看懂代码是什么意思，而是你读懂了别人的思想」**，随着编程经验的积累大部分编码思维方式还是可以读懂和掌握的。

 
相信很多开发者跟我当初一样，只是常用功能点的编写，形成了固定的思维习惯，在编码中才显得得心应手；但在不同的实战上具体的业务逻辑处理「应用场景、优化选择」上却依然有些模糊。
